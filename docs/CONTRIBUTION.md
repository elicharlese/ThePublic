# Contribution Guidelines
## How to Contribute
1. **Fork the Repository**: Click the "Fork" button at the top-right corner of the repository page.
2. **Clone the Forked Repository**:
   ```bash
   git clone https://github.com/your-username/your-repo.git
   cd your-repo

Create a New Branch:

```bash
git checkout -b feature/your-feature-name
```

Make Changes: Implement your feature, bug fix, or hotfix based on immediate feedback.

## Commit Changes:

```bash
git add .
git commit -m "Add detailed message about changes"
```

Push to Your Fork:

```bash
git push origin feature/your-feature-name
```

Create a Pull Request: Go to the original repository and open a pull request.

## Code Style

Follow the existing code style.
Use meaningful variable and function names.
Write comments where necessary.
Write tests for new features or bug fixes.

## Reporting Issues, Feedback & Feature Requests
Before reporting a new issue, make sure it hasn't been discussed already.
Provide detailed information about the issue and steps to reproduce it.
For immediate feedback, use the feedback collection form.
To request a new feature, open an issue and describe the feature in detail, including its benefits and potential use cases.

## Code of Conduct
- Be respectful and considerate of others.
- Avoid inflammatory or offensive language.

## Hotfixes
For critical issues, create an issue and start working on a hotfix as soon as possible.
Document the hotfix properly and provide detailed information in your pull request.

Thank you for contributing!
